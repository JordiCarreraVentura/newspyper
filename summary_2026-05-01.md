## litgpt
**Remote:** https://github.com/Lightning-AI/litgpt/

### Summary of Changes for `litgpt` Repository

The recent commit in the `litgpt` repository focuses on improving the `sample()` function for generating language token predictions. 

#### Key Changes:
- **Greedy Decoding Enhancement**: The `sample()` function now ensures that when the `temperature` is set to 0, it consistently produces greedy (argmax) decoding results. This behavior holds true regardless of the `top_p` value. This change was implemented in both `litgpt/generate/base.py` and `litgpt/generate/speculative_decoding.py` files by altering conditional logic to use `and` instead of `or` for the `temperature` and `top_p` checks.
  
- **New Tests Added**: Two new regression tests have been introduced:
  - `test_sample_temperature_zero_is_greedy()`: Verifies that using a `temperature` of 0 results in greedy decoding across various `top_p` values.
  - `test_sample_top_p_zero_is_greedy()`: Confirms that setting `top_p` to 0 ensures greedy decoding across different `temperature` settings.
  
These changes fix the issue reported in [issue #2238](https://github.com/Lightning-AI/litgpt/issues/2238), ensuring more predictable and correct behavior for applications requiring specific decoding logic. This improvement enhances the model's usability when deterministic outputs are desired.
